Interactive Demo: Impact of Compiler Bugs on OS Kernels

Paper ID: 1234

Abstract

We present supplementary results and tools for our empirical study on how compiler bugs affect OS kernel development, based on analysis of 494 workarounds from 7 popular kernels.

Research Questions & Results

RQ1: Who reports the compiler bugs?

We compare the identity of OS kernel workaround authors and compiler bug reporters to understand knowledge flow.

RQ2: How long does it take from reporting bugs to implementing workarounds?

We analyze the latency between bug report submission and workaround commits.

RQ3: How similar are code samples in bug reports to OS workaround code?

We measure code similarity using JPlag and present distribution results by bug symptom.

RQ4: What is the relationship between symptoms, components, and modified lines?

We link bug symptoms to modified OS components and analyze how symptoms affect code size.
